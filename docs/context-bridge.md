# Context Bridge

## Project Overview
- `pages/01_2026 P&L Projection.py`는 2025년 Won 딜 데이터를 기반으로 2026년 매출/OP를 예측하는 단일 스트림릿 페이지.
- 데이터는 기본적으로 `load_won_deal()`(억 단위 변환)에서 불러오며, 업로드 파일로 대체할 수도 있음.
- 모듈 구성: A) Backlog, B) 삼성 월 계획, C1) 온라인 리텐션(+365), C2) 출강 Upsell(P-tier 배수), D) 목표 대비 신규 딜.
- 모든 매출은 일별 안분 후 월별·연간 집계하며, 고정비(제작·마케팅·인건비·임대·기타)까지 반영한 OP/Rule-of-50 산출.

## UI Structure
1. **데이터 요약**: 전처리 샘플, 기업 규모×채널 피벗, 상위채널×기업규모×S-tier Median, 과정포맷→카테고리 리드타임 표.
2. **시뮬레이션 결과**: KPI metric, 모듈별 기여, 월별 P&L(체결액/매출/공헌/고정비) 표, 연간 P&L 및 고정비 상세.
3. **딜 상세 & 다운로드**: 모듈별 생성된 모든 딜(체결일/교육기간/매출)과 CSV 다운로드.
4. **시뮬레이션 로직**: 데이터/모듈/P&L 계산 로직 상세 설명.

## Key Implementation Notes
- `(온라인)최초 입과 여부=False`인 온라인 딜은 전처리 단계에서 제외.
- 리드타임 = `수주 예정일(종합)` − `생성 날짜`, 교육기간 = `수강종료일` − `수강시작일` + 1.
- Median lookup은 상위채널×기업규모×S-tier → 상위채널×기업규모 → 상위채널 순으로 fallback, 샘플 5건 미만 시 상위 레벨 적용.
- 삼성 계획은 월별 입력값(온라인/출강 각각 1·2.5억 기본)을 12개월 반복.
- 신규 딜은 기업 규모별 2025 매출 비중과 채널별 계절 가중치(Q1~Q4)로 Gap을 월/규모 단위로 배분 후 Median 리드타임·교육기간을 적용.
- 월별 P&L 표에는 체결액, 채널별 매출/공헌, 고정비 세부 항목(제작/마케팅/인건비/임대/기타), OP가 모두 포함.

## Pending / Follow-ups
- 실데이터를 넣어 각 모듈(특히 Backlog/Retention/Upsell)이 0이 아닌지 검증하고 필요 시 컬럼 명이나 필터 재조정.
- Median lookup 샘플 수가 부족한 조합에 대한 기본값/가중치 확인.
- 신규/Upsell 딜 결과가 목표 매출을 정확히 충족하는지, 기업 규모별 분배가 기대와 일치하는지 추가 검증 필요.
